
### 第一步：准备好你的文件
在你的电脑上，确保这些文件和文件夹在**同一个目录**下：
1. `index.html` （必须叫这个名字）
2. `data.js`
3. `images` 文件夹 （里面装着你所有的图片，如 `img_0.jpg` 等）

---

### 第二步：在 GitHub 创建专属仓库

1. 登录你的 [GitHub](https://github.com/) 账号。
2. 点击右上角的 **`+`** 号，选择 **New repository**（新建仓库）。
3. **填写仓库名称 (Repository name)：**
   如果你想把它作为纯正的个人主页，仓库名字**必须严格按照这个格式**：
    `你的GitHub用户名.github.io`
   *(举个例子：如果你的账号名是 `chen123`，那么仓库名字必须填 `chen123.github.io`)*
4. 勾选 **Public**（公开仓库）。
5. 勾选 **Add a README file**（添加自述文件，方便一会直接上传代码）。
6. 点击最下方的绿按钮 **Create repository**（创建仓库）。

---

### 第三步：上传你的代码和图片

1. 进入你刚创建好的仓库页面，点击中间偏右的 **Add file** 按钮，选择 **Upload files**（上传文件）。
2. 把你电脑上的 `index.html`、`data.js` 和 `images` 文件夹，**全部拖拽**到网页的上传框里。
3. 等待网页上的进度条跑完（确保 `images` 里面的图片也都传上去了）。
4. 传完后，在下方点击绿色的 **Commit changes**（提交更改）按钮。

> **检查一下**：此时你的仓库首页应该能看到 `index.html`、`data.js` 以及 `images` 文件夹并列在一起。

---

### 第四步：开启 GitHub Pages 服务

*(如果你的仓库名字严格按 `用户名.github.io` 命名的，GitHub 通常会自动帮你开启，但我们最好检查一下)*

1. 在你的仓库页面，点击顶部的 **Settings**（设置）齿轮图标。
2. 在左侧菜单栏往下拉，找到 **Pages**（页面）选项并点击。
3. 在 **Build and deployment** 下方的 **Source**，确认选择的是 **Deploy from a branch**。
4. 在 **Branch** 下方的下拉菜单中，选择 **main** （或者 master），后面的文件夹保留默认的 **`/ (root)`**。
5. 点击 **Save**（保存）。

---

### 第五步：访问你的主页

1. 刚刚保存完后，GitHub 需要大约 **1到3分钟** 的时间来为你构建网页。
2. 你可以在仓库主页点击黄色的圆点（如果有的话）查看进度，变成绿色对勾说明部署完毕。
3. 打开浏览器，输入你的专属网址：
    `https://你的用户名.github.io`
   

---

### 后续维护和更新数据

因为已经把数据和前端代码分离开了，以后你上新产品或者修改文本，

1. 登录 GitHub，进入你的这个仓库。
2. 点击打开 `data.js` 文件。
3. 点击右上角的 **✏️ (铅笔图标)** 进入编辑模式。
4. 直接在网页上修改文本或添加新产品的 JSON 数据。
5. 改完后，点击右上角的 **Commit changes** 保存。
6. 等待 1 分钟左右，刷新你的网站，新内容就上线了！

如果有新的图片，就点进 `images` 文件夹，选择 `Add file -> Upload files` 把新图片传上去，然后在 `data.js` 里写上图片的名字就行。
